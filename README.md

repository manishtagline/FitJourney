# FitJourney

[![Build status](https://img.shields.io/badge/build-pending-lightgrey)]() [![License: MIT](https://img.shields.io/badge/license-MIT-blue)]()

A Spring Boot fitness-tracking application to log workouts, meals, health metrics and track progress over time. FitJourney provides endpoints for user registration/authentication (JWT), workout and meal logging, health metrics, and goal tracking.

## Quickstart ‚Äî run locally
1. Clone the repo
   git clone https://github.com/manishtagline/FitJourney.git
   cd FitJourney

2. Configure the database
   - Create a database (example: `fitjourney_db`).
   - Copy and edit `src/main/resources/application.properties` or create `application-local.properties` to point to your DB (sample below).

3. Build and run
   mvn clean package
   mvn spring-boot:run

   Or run the generated jar:
   java -jar target/fitjourney-0.0.1-SNAPSHOT.jar

## Configuration (example)
In `src/main/resources/application.properties` or via environment variables:

spring.datasource.url=jdbc:mysql://localhost:3306/fitjourney_db?useSSL=false&serverTimezone=UTC
spring.datasource.username=your_db_user
spring.datasource.password=your_db_password
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true

# JWT config (example)
app.jwt.secret=replace_this_with_a_secure_random_value
app.jwt.expirationMs=86400000

Note: For production, store secrets in environment variables or a secrets manager; do NOT commit them to the repo.

## Run with Docker Compose (optional)
Create a simple `docker-compose.yml` with app + MySQL, or use the following pattern:

version: '3.8'
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: fitjourney_db
      MYSQL_USER: fituser
      MYSQL_PASSWORD: fitpass
      MYSQL_ROOT_PASSWORD: rootpass
    ports:
      - "3306:3306"
    volumes:
      - db-data:/var/lib/mysql
volumes:
  db-data:

(Then configure the app datasource to point to `db:3306`.)

## API basics and auth
- Base URL: http://localhost:8080 (by default)
- Auth endpoints (examples):
  - POST /auth/register ‚Äî register new user (accepts JSON with username, email, password, profile fields)
  - POST /auth/login ‚Äî login and receive JWT token

Example curl (login):
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"secret"}'

Response:
{
  "token": "<JWT_TOKEN>",
  "type": "Bearer",
  "username": "alice"
}

Protected endpoints require header:
Authorization: Bearer <JWT_TOKEN>

Document your endpoints and request/response shapes ‚Äî adding Swagger/OpenAPI will help users and front-end developers.
